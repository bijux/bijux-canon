# Observability Modules

The runtime observability surface is split into four code-facing groups:

## Analysis

- `trace_diff`
- `comparative_analysis`
- `drift`
- `flow_invariants`
- `flow_correlation`

## Capture

- `trace_recorder`
- `observed_run`
- `hooks`
- `time`
- `environment`
- runtime observers

## Classification

- `determinism_classification`
- `entropy`
- `fingerprint`
- `seed`
- `retrieval_fingerprint`

## Storage

- `execution_store`
- `execution_store_protocol`
- `schema_contracts`
- `migrations`
- `schema.sql`
- `schema.hash`
