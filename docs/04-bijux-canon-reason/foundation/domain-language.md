# Domain Language

The language around `bijux-canon-reason` should make reasoning policy explicit instead of letting it hide behind workflow or prompt language. Reviewable meaning needs stable nouns.

## Prefer These Terms

- say `claim`, `verification`, `evidence`, and `provenance` when reasoning owns the decision
- name artifacts by the reasoning judgment they carry
- use `reasoning step` for local evaluation behavior before orchestration begins

## Avoid These Terms

- avoid using retrieval words for reasoning judgments
- avoid calling orchestration or runtime policy “reasoning” just because it happens later
- avoid soft labels like `AI output` where a precise reasoning noun exists

## Bottom Line

If the vocabulary makes package ownership harder to see, the vocabulary is wrong even if the sentence sounds smooth.
